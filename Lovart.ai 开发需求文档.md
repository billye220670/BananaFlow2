

# Lovart.ai 开发需求文档

## 1. 项目概述

### 1.1 产品定位
Lovart.ai 是一个基于 Nano Banana AI 图像生成技术的创意设计平台，通过自然语言对话驱动图像创作和编辑，为用户提供零门槛的设计体验。

### 1.2 核心价值
- **对话驱动**: 通过自然语言指令控制所有设计操作
- **统一画布**: 单一工作区承载所有创作内容
- **智能编辑**: 支持拖拽图片进行AI增强和风格迁移
- **多图参考**: 可上传多张参考图指导AI生成

## 2. 系统架构

### 2.1 整体布局
采用**双面板响应式布局**：
- **左侧主画布 (70%)**: 统一工作区，支持图像展示和拖拽交互
- **右侧聊天面板 (30%)**: 对话界面，处理用户指令和参考图管理

### 2.2 技术栈
- **前端**: React + Tailwind CSS
- **状态管理**: React Context 或 Zustand
- **API 集成**: FAL.ai Nano Banana API
- **文件处理**: FileReader API + 临时存储服务
- **响应式**: 移动优先，支持桌面/平板/手机

## 3. 核心功能需求

### 3.1 主画布功能

#### 3.1.1 基础功能
- **拖拽支持**: 接受用户从桌面拖拽图片文件
- **单图显示**: 同时只显示一张主要图像（最新生成或编辑结果）
- **全屏模式**: 支持切换到全屏专注查看
- **导出功能**: 支持 PNG/JPG 格式下载

#### 3.1.2 编辑模式
- **自动激活**: 检测到拖入图片时自动进入编辑模式
- **状态指示**: 显示当前编辑状态（"编辑中..."、"完成"等）
- **上下文菜单**: 右键点击图片显示编辑选项
- **版本对比**: 支持查看编辑前后对比（可选）

### 3.2 聊天面板功能

#### 3.2.1 对话系统
- **消息历史**: 显示用户指令和AI响应的历史记录
- **智能输入**: 根据当前模式显示不同的占位符文本
- **发送机制**: 支持 Enter 发送，Ctrl+Enter 换行
- **加载状态**: 显示 "AI正在处理..." 状态

#### 3.2.2 参考图管理
- **附件栏**: 在输入框上方显示参考图缩略图
- **上传方式**: 支持拖拽和点击上传
- **数量限制**: 最多6张参考图（Nano Banana API 限制）
- **格式支持**: JPG, PNG, WebP
- **文件大小**: 单文件 ≤ 10MB，总大小 ≤ 50MB

#### 3.2.3 智能上下文
- **模式检测**: 自动识别创建模式 vs 编辑模式
- **建议指令**: 根据当前状态提供常用指令建议
- **目标锁定**: 编辑模式下自动关联主画布图片

## 4. 交互流程设计

### 4.1 创建模式流程
```
1. 用户在聊天面板输入创作指令
   ↓
2. 系统调用 Nano Banana API 生成图像
   ↓
3. 结果显示在主画布上
   ↓
4. 用户可继续输入修改指令或导出结果
```

### 4.2 编辑模式流程
```
1. 用户拖拽图片到主画布
   ↓
2. 系统自动激活编辑模式，锁定编辑目标
   ↓
3. 用户可选择上传参考图（可选）
   ↓
4. 用户输入编辑指令
   ↓
5. 系统调用 API 进行图像编辑
   ↓
6. 编辑结果替换主画布原图
```

### 4.3 混合工作流
用户可在创建和编辑模式间自由切换：
- 从创建模式生成的图片可直接拖拽回画布进入编辑模式
- 编辑完成的图片可作为新创作的起点

## 5. API 集成规范

### 5.1 Nano Banana API 调用

#### 5.1.1 创建模式调用
```javascript
{
  prompt: "用户描述的创作指令",
  num_images: 1,
  aspect_ratio: "auto",
  output_format: "png",
  image_urls: [参考图URL数组], // 可选
  resolution: "1K"
}
```

#### 5.1.2 编辑模式调用
```javascript
{
  prompt: "用户描述的编辑指令",
  num_images: 1,
  aspect_ratio: "auto",
  output_format: "png",
  image_urls: [目标图片URL, ...参考图URL数组],
  resolution: "1K"
}
```

### 5.2 文件处理流程
1. **前端预览**: 使用 FileReader 生成本地预览
2. **临时上传**: 将文件上传到临时存储服务获取可访问URL
3. **API 调用**: 将URL传递给 Nano Banana API
4. **清理机制**: 设置临时文件过期时间（24小时）

## 6. 状态管理设计

### 6.1 核心状态变量
| 状态 | 类型 | 描述 |
|------|------|------|
| `canvasImage` | string/null | 当前主画布显示的图像URL |
| `chatHistory` | array | 聊天消息历史记录 |
| `referenceImages` | array | 已上传的参考图File对象数组 |
| `isEditingMode` | boolean | 是否处于编辑模式 |
| `editingTarget` | File/null | 当前编辑目标图片 |
| `isLoading` | boolean | API调用加载状态 |

### 6.2 状态转换规则
- 拖入新图片 → `isEditingMode = true`, `editingTarget = 新图片`
- 清空画布 → `isEditingMode = false`, `editingTarget = null`
- 上传参考图 → 更新 `referenceImages` 数组
- API调用开始 → `isLoading = true`
- API调用完成 → `isLoading = false`, 更新 `canvasImage`

## 7. 响应式设计规范

### 7.1 断点定义
- **桌面端**: ≥ 1024px - 标准双面板布局
- **平板端**: 768px - 1023px - 调整面板比例
- **移动端**: < 768px - 垂直堆叠布局

### 7.2 移动端特殊处理
- **默认视图**: 显示主画布，聊天面板可滑动展开
- **输入优化**: 底部固定输入区域，避免键盘遮挡
- **触摸交互**: 长按图片进入编辑模式
- **简化功能**: 优先展示快速编辑选项

## 8. 错误处理与用户体验

### 8.1 错误类型与处理
| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 文件格式错误 | 拒绝上传 | "仅支持 JPG/PNG/WebP 格式" |
| 文件大小超限 | 拒绝上传 | "文件大小不能超过 10MB" |
| 参考图数量超限 | 阻止上传 | "最多支持 6 张参考图" |
| API 调用失败 | 保持当前状态 | "生成失败，请重试" |
| 网络错误 | 提供重试按钮 | "网络连接失败，请检查网络" |

### 8.2 加载状态反馈
- **文件上传**: 显示进度条和文件名
- **API 调用**: 显示旋转动画 + "AI正在思考..."
- **生成完成**: 平滑过渡动画更新画布

## 9. 性能要求

### 9.1 性能指标
- **界面响应**: 所有交互响应时间 ≤ 100ms
- **图像生成**: API 调用完成时间 ≤ 30秒
- **文件上传**: 10MB 文件上传时间 ≤ 5秒
- **内存占用**: 单页面内存占用 ≤ 200MB

### 9.2 优化策略
- **懒加载**: 参考图缩略图按需加载
- **缓存机制**: 已上传文件本地缓存
- **并行处理**: 多文件并行上传
- **内存清理**: 及时释放不再使用的图像数据

## 10. 安全与隐私

### 10.1 数据安全
- **传输加密**: 所有API调用使用HTTPS
- **API密钥**: 前端不直接暴露API密钥，通过代理服务
- **文件安全**: 临时文件设置访问权限和过期时间

### 10.2 内容安全
- **API过滤**: 启用Nano Banana的安全过滤（safety_tolerance: "4"）
- **用户提示**: 不当内容生成时显示友好提示
- **日志记录**: 记录异常请求用于安全分析
